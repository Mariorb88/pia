from sklearn.pipeline import Pipeline
from sklearn.preprocessing import StandardScaler, OneHotEncoder
from sklearn.compose import ColumnTransformer
import pandas as pd

def get_preprocessor(df):
    num_attribs = df.select_dtypes(include=["float64", "int"]).columns.drop("median_house_value")
    cat_attribs = df.select_dtypes(include=["object"]).columns

    num_pipeline = Pipeline([
        ('std_scaler', StandardScaler())
    ])

    full_pipeline = ColumnTransformer([
        ("num", num_pipeline, num_attribs),
        ("cat", OneHotEncoder(handle_unknown='ignore'), cat_attribs),
    ])

    return full_pipeline

    keras.layers.Dense(1)

history = model.fit(X_train_prep, y_train, epochs=50, validation_data=(X_valid_prep, y_valid))

# Evaluar

Un ensamble de una red neuronal con XGBoost es una tcnica que combina las predicciones de ambos modelos para obtener un resultado ms preciso que el que lograran por separado.

Cmo funciona el ensamble?

La idea es aprovechar lo mejor de dos mundos:

ModeloFortalezas principales
Red neuronalCaptura relaciones complejas y no lineales en los datos. Muy flexible.
XGBoostMuy eficaz en datos tabulares, detecta relaciones jerrquicas y robusto frente a valores atpicos.

En un ensamble, combinamos ambas predicciones con un peso :

python
pred_final =  * pred_nn + (1 - ) * pred_xgb

Donde:

  • pred_nn: prediccin de la red neuronal

  • pred_xgb: prediccin de XGBoost

  • es un valor entre 0 y 1 (por ejemplo, 0.15 significa 15% red neuronal y 85% XGBoost)

Por qu usar un ensamble?

  • Porque cada modelo comete errores distintos.

  • Un buen ensamble reduce el error global (como el RMSE) al suavizar esos errores.

  • Muy til si un modelo es bueno en un rango de datos y otro en otro (por ejemplo, NN en patrones complejos, XGBoost en valores extremos).

Buena pregunta. Para un dataset como California Housing (pequeo, tabular, con pocas features), PyTorch y TensorFlow son tcnicamente equivalentes en cuanto a capacidad de modelado, pero hay algunas diferencias prcticas que pueden hacer que uno sea ms adecuado que otro segn el caso.

Dataset California Housing

  • Tamao pequeo (20k ejemplos)

  • Tarea de regresin

  • Ideal para probar modelos simples/medios en MLP (redes densas)

Cul es mejor en este caso?

CriterioPyTorchTensorFlow
Simplicidad y control manual Ms claro, muy pythonicMs verboso, pero estructurado
Visualizacin (integrada)Menos nativa TensorBoard + Keras History
Tuning automtico (KerasTuner)Menos integrado Fcil y potente

Entonces?

  • Si te interesa el control detallado del entrenamiento y quieres aprender cmo funciona todo paso a paso, PyTorch es ideal (como lo ests haciendo ahora).

  • Si te interesa una experiencia ms automatizada, rpida y modular (con callbacks, logs, tuning), TensorFlow + Keras puede ser ms cmodo.
